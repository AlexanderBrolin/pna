# Process Network Analyzer (PNA)

Десктопное приложение для мониторинга сетевой активности процессов на Windows. Показывает все TCP и UDP соединения выбранного приложения с резолвом IP-адресов в доменные имена. Полезно для создания точечных маршрутов (KnockDNS, Antizapret) и анализа сетевого поведения приложений.

## Возможности

- **Мониторинг по процессу или пулу процессов** — выбираешь `chrome.exe` и автоматически отслеживаются ВСЕ процессы Chrome (вкладки, расширения, service workers). Новые процессы добавляются в пул автоматически.
- **TCP-соединения** — захват через `GetExtendedTcpTable` с поллингом 20мс. Ловит даже короткоживущие соединения (HTTP-запросы, API-вызовы).
- **UDP-соединения** — захват через raw socket (`SIO_RCVALL`) + привязка к процессу через `GetExtendedUdpTable`. Ловит VoIP-звонки (Telegram, Discord), QUIC-трафик (YouTube, Google), DNS-запросы.
- **Резолв доменов** — три источника: ETW DNS-Client (перехват DNS-запросов с привязкой к PID), кэш DNS Windows (`ipconfig /displaydns`), reverse DNS (fallback).
- **Заблокированные соединения** — видны даже соединения, заблокированные DPI/ТСПУ: DNS-запрос фиксируется до попытки соединения, TCP SYN_SENT ловится поллингом, UDP-пакеты перехватываются на уровне интерфейса.
- **Экспорт** — домены, IP-адреса (с агрегацией подсетей /24), JSON с полной метаинформацией.
- **Детекция VPN-туннеля** — автоматическое определение, идёт ли трафик через VPN-туннель (по IP-диапазонам). Настраиваемые подсети туннеля (по умолчанию 10.29.0.0/16, 10.30.0.0/15 для AntiZapret-VPN).
- **Детекция проблемных соединений** — отслеживание TCP-соединений в состоянии SYN_SENT. Если соединение не устанавливается в течение 5 секунд — строка подсвечивается красным. Помогает выявить блокировки DPI/ТСПУ.
- **Чёрный список** — встроенный фильтр системных доменов (Microsoft, Windows Update, OCSP и т.д.).

## Как это работает

### Архитектура

```
┌─────────────────────────────────────────────────────┐
│  Electron Frontend (HTML/JS)                        │
│  ← WebSocket →                                      │
│  Python Backend (asyncio + threads)                 │
│                                                     │
│  ┌──────────────┐  ┌───────────────┐  ┌───────────┐ │
│  │ DNS ETW      │  │ TCP Poller    │  │ UDP Raw   │ │
│  │ (domain→IP)  │  │ (20ms, ctypes)│  │ Socket    │ │
│  └──────┬───────┘  └──────┬────────┘  └──────┬────┘ │
│         └──────────┬──────┘                  │      │
│              ┌─────▼──────┐    ┌─────────────▼─┐    │
│              │ Aggregator │    │ UDP Table     │    │
│              │ (dedup,    │    │ (port→PID map)│    │
│              │  merge)    │    └───────────────┘    │
│              └─────┬──────┘                         │
│              ┌─────▼───────┐                        │
│              │ rDNS / DNS  │                        │
│              │ Cache       │                        │
│              └─────────────┘                        │
└─────────────────────────────────────────────────────┘
```

### Механизмы захвата

| Протокол | Метод | Привязка к PID | Интервал |
|----------|-------|----------------|----------|
| TCP | `GetExtendedTcpTable` (ctypes) | Нативно из WinAPI | 20мс |
| UDP | Raw socket `SIO_RCVALL` + `GetExtendedUdpTable` | Через local_port→PID таблицу | Real-time + 200мс |
| DNS | ETW `Microsoft-Windows-DNS-Client` | Нативно (ClientPID) | Event-driven |

### Резолв IP → Домен (приоритет)

1. **ETW DNS** — перехват DNS-запросов процесса в реальном времени (домен + IP + PID)
2. **DNS Cache Windows** — `ipconfig /displaydns` каждые 2 сек (все DNS-запросы системы)
3. **Reverse DNS** — `gethostbyaddr()` для IP без домена (8 параллельных потоков, таймаут 3 сек)

## Требования

- Windows 10/11
- Python 3.10+
- Права администратора (для ETW и raw socket)

## Установка

### Из релиза (рекомендуется)

Скачайте установщик из [Releases](https://github.com/AlexanderBrolin/pna/releases), запустите, следуйте инструкциям.

### Из исходников

```bash
# Клонировать
git clone https://github.com/AlexanderBrolin/pna.git
cd pna

# Установить зависимости
npm install
pip install -r backend/requirements.txt

# Запустить в dev-режиме
npm start

# Собрать exe
npm run build-exe
```

## Использование

1. Запустите приложение **от имени администратора** — это обязательно для ETW-трассировки и raw socket.
2. В поле поиска начните вводить имя процесса (например, `chrome`). Появится выпадающий список с группировкой по имени — можно выбрать один процесс или сразу весь пул.
3. Нажмите **Начать запись**. В правом верхнем углу появится зелёный индикатор «Мониторинг» — значит, система активно отслеживает соединения.
4. **Начните пользоваться приложением** — откройте сайты, позвоните, используйте нужные функции. Соединения появляются в таблице по мере их обнаружения.

> **Важно:** соединения обнаруживаются не мгновенно. TCP-поллинг работает с интервалом 20мс, но некоторые очень короткоживущие соединения могут быть пропущены при первом проходе. Для полноты картины рекомендуется поработать с приложением некоторое время и повторить нужные действия несколько раз. DNS-запросы ловятся надёжнее — через ETW они фиксируются event-driven, до установления соединения.

5. Используйте **поиск по таблице**, **сортировку** по столбцам (клик на заголовок) и **фильтры** для анализа.
6. В панели **Фильтры** (кнопка справа) можно:
   - Скрыть туннелированные соединения (те, что уже идут через VPN)
   - Настроить подсети туннеля (по умолчанию AntiZapret: 10.29.0.0/16, 10.30.0.0/15)
   - Добавить домены в чёрный список (например, `*.microsoft.com`)
7. Строки, подсвеченные **красным** — это соединения, которые не удалось установить (TCP SYN_SENT без ответа дольше 5 секунд). Это может указывать на блокировку DPI/ТСПУ.
8. **Экспортируйте** результат: домены, IP-адреса (с агрегацией в подсети /24) или полный JSON.

### Применение для маршрутизации (Antizapret / KnockDNS)

1. Запустите мониторинг нужного приложения (браузер, мессенджер и т.д.)
2. Используйте все функции приложения — откройте разные разделы сайта, позвоните, отправьте файл
3. В панели **Фильтры** включите «Скрыть туннелированные» — в таблице останутся только соединения, идущие **напрямую** (мимо VPN)
4. Обратите внимание на красные строки — это потенциальные кандидаты для добавления в маршруты
5. Выберите нужные записи → **Экспорт доменов** → добавьте в конфиг KnockDNS / Antizapret
6. Или: **Экспорт IP/подсетей** → добавьте в AllowedIPs WireGuard/AmneziaWG

## Структура проекта

```
├── backend/
│   ├── server.py           # WebSocket сервер, оркестрация
│   ├── etw_tracer.py       # ETW DNS + TCP poller + UDP raw socket
│   ├── aggregator.py       # Дедупликация, IP→домен, группировка
│   ├── process_tree.py     # Отслеживание процессов и дочерних
│   ├── dns_cache.py        # Поллинг ipconfig /displaydns
│   ├── rdns_resolver.py    # Reverse DNS резолвер
│   ├── blacklist.py        # Чёрный список доменов
│   └── requirements.txt
├── src/
│   ├── main.js             # Electron main process
│   ├── preload.js          # Preload script
│   └── renderer/
│       ├── index.html
│       ├── app.js          # UI логика
│       └── styles.css
└── package.json
```
